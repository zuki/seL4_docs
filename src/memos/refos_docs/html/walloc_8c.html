Allocate a window, with extra options.

Much like walloc(), but with extra control over permission and flag values of the window. eg. used to implement device mappings, which should be mapped uncached.

Parameters
npagesThe size of the window (in number of pages, page size is REFOS_PAGE_SIZE, usually 4096 bytes).
[out]windowOutput cptr to store the allocated window capability, if success. (No ownership)
permissionThe window permission flags, defined in proc_client_helper.h.
flagsControls window flags such as cached/uncached. See proc_client_helper.h.
Returns
VAddr of allocated window if success, 0 otherwise.

◆ walloc_get_window_at_vaddr()

seL4_CPtr walloc_get_window_at_vaddr ( seL4_Word  vaddr)

Retrieve the window capability in the current client's vspace at a given vaddr.

This is a helper function used to avoid the need for double-book-keeping of vspace to window capabilities, used for releasing allocated windows without book-keeping them again. Used for RefOS userland dynamic mmap implementation.

Parameters
vaddrThe virtual address to retrieve window for.
Returns
Copy of capability to window at the given vaddr if any, 0 otherwise. Ownership is given for the cslot and cap, so caller must delete those and release the cslot.
