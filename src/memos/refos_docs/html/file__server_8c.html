Detailed Description

CPIO File Server main source file.

The main role of the fileserver in the system is to provide the executable file contents via the dataspace interface. It does some book-keeping and simple management of clients.

The CPIO file server:

  • Supports providing pager service to clients.
  • Supports providing content-initialisation for another dataserver.
  • Supports parameter buffers using an external dataspace (i.e. a dataspace provided by another dataserver).
  • Does NOT support parameter buffers using an internal dataspace (i.e. a dataspace provided the fileserver itself).
  • Does NOT support having its dataspace content-initalised by an external dataspace.
  • Ignores nBytes parameter in open() method (actual CPIO file size is used).

The CPIO file server provides access to files stored in a CPIO archive inside the server. The CPIO archive is a simple format file archive stored inside a parent program's ELF section. This is a similar idea to something like creating a

const char data[] = { 0x3F, 0xFF, 0x23 ...etc}

The CPIO file server is used to store the RefOS userland app ELF binaries, and any additional data files. It provides a simple example of a dataserver, and may serve as the means to boot more complex dataspace servers involving flash memory / SATA drivers and so forth.
